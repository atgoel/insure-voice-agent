# Implementation Plan: Product Ranking & Recommendation Explainer

**Spec**: specs/004-product-ranking-explainer/spec.md | **Date**: 2026-05-29

## Summary

Sub-Agent 3 calls the `rank_products` Cloud Function to score compliance-passed products with a
weighted suitability formula (ELSER √ó 0.4 + age_centrality √ó 0.3 + income_fit √ó 0.3), returning
the top-3 ranked products with a full audit trail. The Root Agent then uses the top-3 to generate
a voice-optimised (‚Č§120 words, WaveNet-safe) recommendation explanation personalised to the
customer's profile.

## Constitution Check

- [x] **Compliance guardrail respected** ‚ÄĒ `rank_products` only receives `passed_products`; rejected products cannot enter the input. Root Agent prompt explicitly prohibits recommending rejected products (¬ßI).
- [x] **Latency target honoured** ‚ÄĒ Cloud Function is pure Python arithmetic; no I/O, no external calls. P95 well within the 1s budget (¬ßIII).
- [x] **No hallucination risk** ‚ÄĒ Scoring formula is fully deterministic. Explanation is generated by Gemini from factual top-3 data only; no product eligibility decisions are delegated to the LLM (¬ßII).
- [x] **Audit trail** ‚ÄĒ `audit.all_scored` (all evaluated products), `formula_weights`, `customer_profile_hash` (SHA-256, no PII) in every response (¬ßIV, ¬ßV).

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| ELSER score normalisation | Divide by batch max; fall back to `1.0` when max=0 | Prevents the absolute magnitude of ELSER BM25 hybrid scores (often > 1.0) from dominating the weighted formula without masking relative ordering |
| Profile hash | SHA-256 of `json.dumps(profile, sort_keys=True)` | Deterministic across dict insertion order; one-way (¬ßV no PII); 64-char hex is human-readable in Cloud Logging |
| `income_fit` default when `sum_need=0` | `0.5` (neutral) | Avoids punishing customers who haven't stated a sum need; `0.0` would unfairly deprioritise otherwise good products |
| `min_age == max_age` edge case | `age_span = max(max_age - min_age, 1)` | Prevents division-by-zero; customer at exact age gets `age_centrality=1.0`, otherwise `0.0` |
| Explanation generation location | Root Agent (Gemini), not the Cloud Function | Keeps the Cloud Function deterministic and latency-predictable (¬ßII, ¬ßVII). The Cloud Function's job ends at top-3 + audit |
| Sub-Agent 3 prompt scope | Receives `top3` from `rank_products` + customer profile summary; outputs ‚Č§120 word voice script | Matches the "warm advisor" tone described in root_agent_prompt.md; avoids markdown/tables in TTS output |
| All-scored sort order in audit | Descending by `suitability_score` | Consistent with top-3 ordering; simplifies debugging |

The Sub-Agent 3 system prompt must specify:
